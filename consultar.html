Consultar una actividad

Endpoint GET /api/actividades/{id} — El formulario recoge el parámetro de ruta id. La tabla muestra el ActividadResponse retornado.

GET

Entrada — Path variable: {id}

El único parámetro es el id de la actividad, que va en la URL como variable de ruta: /api/actividades/{id}. Es un Long positivo asignado por la base de datos al crear la actividad.

Long · @PathVariable · va en la URL: /api/actividades/{id}
Ver todas
200 OK

Respuesta — Body: ActividadResponse

Si el id existe, el backend retorna el objeto completo con HTTP 200. Si no existe, retorna HTTP 404 con un ErrorResponse estructurado.
